# Interaction Protocol

Use the ToK as a context resource without collapsing its distinctions.

## Query separation

Every query must be able to include or exclude these jurisdictions independently:

- `content`
- `logia/general`
- `logia/librarian`

Default knowledge queries include `content` and may include `logia/general` as guidance. They exclude `logia/librarian`.

## Interpretation

1. Preserve source material before interpreting it.
2. Distinguish an author's statement from a librarian's extrapolation.
3. Do not manufacture definitions, relations, consensus, or readiness.
4. Treat placement and wording as proposals until ratified.
5. Preserve stable identity through renaming, grafting, pruning, and relocation.
6. Return uncertainty and unresolved alternatives with the entry.
7. Prefer filtered context compilation over indiscriminate synthesis.

## Entry envelope

An entry may carry:

- stable identity
- kind
- source and provenance
- meaning
- scope and context
- placement
- relations
- readiness
- status
- ratification record

Absent properties remain unknown rather than being inferred as fact.

## Structural operations

- A trim changes the context returned without deleting canonical content.
- A graft changes placement or relation while preserving identity and history.
- A rename changes the label while preserving identity and history.
- A prune removes a relation or placement through a reviewed, traceable proposal.
