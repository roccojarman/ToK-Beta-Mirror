# Repository Architecture

Status: provisional architecture, committed for use and revision.

## Repository family

| Repository | Jurisdiction |
|---|---|
| ToK-Beta-Kernel | Kernel source |
| ToK-Beta-Acorn | Acorn source |
| ToK-Beta-Tree | Tree source |
| ToK-Beta-Mirror | Writable working reflection and curation workspace |
| ToK-Beta-WholeTree | Read-only impression assembled from the three canonical sources |

The wider repository family is planned. This Mirror architecture does not assert that every listed repository already exists.

## Planes

`Content/` holds knowledge entries and their structural placement.

`Logia/General/` holds architecture, governance, vocabulary, context, and instructions for people and agents using the ToK.

`Logia/Librarian/` holds Mirror-specific operational context, instructions, precedents, protocol, and working state for librarians. Librarian material must not be returned as knowledge content unless explicitly requested.

## Content topology

- Kernel holds universal ordering principles.
- Acorn holds durable principles and aphorisms.
- Tree holds Bridge-or-Trunk universal context and domain-specific Roots-and-Branches.
- `Bridge-or-Trunk` is an unresolved name and must remain unchanged until ratified.
- Roots and Branches is one structural category. A root may operate as a branch and a branch as a root according to relation and context.

## Authority boundary

The Mirror may contain proposals and working representations. It is not authority to alter canonical sources. Promotion requires an explicit gateway and ratification appropriate to the destination repository.

## Impression flow

Canonical Kernel, Acorn, and Tree sources are re-impressed into WholeTree through an explicit workflow. Mirror informs proposals but is not itself a canonical source.
